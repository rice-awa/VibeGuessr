## 阶段二：后端核心 API

### 2.1 基础架构
- [x] Flask 应用入口 & 路由注册（app.py）
- [x] 配置管理（config.py：API 密钥、难度参数表）
- [x] SQLite 数据库初始化 & 数据模型（game.py）
- [x] 游戏会话管理（session_id 生成与存储）

### 2.2 LLM 服务（llm_service.py）
- [x] Gemini Chat API 调用封装（OpenAI 兼容格式）
- [x] 出词 Prompt 模板（word_gen.py）：按难度生成关键词 + 视觉描述 + 提示语
- [x] 判分 Prompt 模板（judge.py）：结构化判定答案匹配度
- [x] 知识卡片生成：猜对后返回趣味冷知识

### 2.3 图片服务（image_service.py）
- [x] GPT-image-2 调用封装（模糊图 / 清晰图）
- [x] 不同难度的 Prompt 策略（模糊程度控制）
- [x] 图片 Base64 / URL 响应处理

### 2.4 游戏逻辑（game_service.py）
- [x] 得分计算（基础分 × score_ratio × 时间奖励 × 提示惩罚）
- [x] 连续答对加成（3题+10%、5题+20%、10题+50%）
- [x] 已出词去重（上下文传递）
- [x] 每题最多 3 次作答限制

### 2.5 API 端点
- [x] `POST /api/game/start` — 开始新游戏
- [x] `POST /api/game/next` — 获取下一题（关键词 + 模糊图片）
- [x] `POST /api/game/guess` — 提交答案，返回判分
- [x] `POST /api/game/hint` — 获取文字提示
- [x] `POST /api/game/reveal` — 猜对后获取清晰图 + 知识卡片
- [x] `GET /api/game/result` — 本局结算数据
- [x] 错误处理 & 超时重试（15 秒超时、重试 1 次、降级跳过）

## 阶段三：前端页面 & 交互

### 3.1 页面开发
- [x] **首页**：Logo、开始游戏按钮、模式选择入口
- [x] **难度选择页**：三个难度卡片，展示规则差异（词库范围、提示次数、时限、分值）
- [x] **游戏主页**：模糊图片展示、答案输入框、倒计时、提示按钮、当前得分、题号进度
- [x] **答题反馈弹窗**：判定结果（exact/close/related/wrong/timeout）、得分、反馈语、知识卡片、清晰图片
- [x] **结算页**：总分、正确率、平均用时、最佳答题回顾、答题记录、再来一局按钮

### 3.2 核心组件
- [x] Timer 倒计时组件（SVG 圆环进度条，支持不同难度时限，颜色随时间变化）
- [x] ScoreBar 得分展示组件（顶栏 pill 样式）
- [x] HintButton 提示按钮（显示剩余次数，禁用态处理）
- [x] ImageDisplay 图片展示组件（骨架屏加载、淡入效果）
- [x] AnswerInput 答案输入组件（表单提交、回车提交）
- [x] FeedbackModal 答题反馈弹窗（5 种判定状态视觉区分、清晰图展示、知识卡片、重试/下一题）
- [x] LoadingOverlay 等待动画（点阵跳动 + 上下文提示文字）

### 3.3 自定义 Hooks
- [x] useTimer — 倒计时逻辑（已有）
- [x] useGame — 游戏状态管理（完整状态机：idle→starting→loading→playing→judging→feedback→finished）

### 3.4 API 调用层（services/）
- [x] 封装所有后端 API 调用（已有）
- [x] 请求 / 响应错误处理（useGame 中统一处理）

### 3.5 状态管理（store/）
- [x] 游戏全局状态（useGame hook 管理：难度、session_id、当前题目、积分）
- [x] localStorage 持久化积分数据（已有）
