# PRD优化总结

## 任务概述
基于用户提供的 `prd-demo.md` 初稿，结合API技术文档和用户反馈，优化并输出完整的产品需求文档 `PRD.md`。

## 主要优化内容

### 相比原始方案新增的部分
1. **智能判分系统**：定义了同义词100%得分、近义词60-80%、沾边20-40%的分级评分规则，包含完整的Prompt设计和few-shot示例
2. **得分计算公式**：引入时间奖励系数和提示惩罚系数，让评分更有层次
3. **提示系统**：3级递进文字提示 + 图片清晰化提示，解决玩家卡住的问题
4. **倒计时机制**：不同难度不同时限（30/45/60秒），控制游戏节奏
5. **性能优化策略**：预加载下一题、并行请求、降级方案，处理图片生成5-15秒延迟
6. **防作弊与边界处理**：覆盖图片暴露答案、恶意提交、API超时等异常场景
7. **MVP范围界定**：三期迭代计划，MVP聚焦单人模式核心循环
8. **成本预估**：每局约47次LLM调用 + 17次图片生成
9. **前端页面结构**：5个页面 + 游戏主页布局线框图

### 技术方案确认
- LLM：Gemini（OpenAI兼容格式，gemini-2.5-flash）
- 图片生成：GPT-image-2-all
- 图片理解：Gemini（备用校验）
- 图片编辑：GPT-image-1.5（备用）
- 前端：原生HTML/CSS/JS，MVP无需后端数据库

## 输出文件
- `PRD.md` — 完整产品需求文档
